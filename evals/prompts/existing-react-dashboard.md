# Eval Prompt: Existing React Dashboard

Use `$noootwo-design` to redesign an existing React dashboard that feels generic and inefficient.

Expected behavior:

- Capture existing system, tokens, components, data surfaces, and screenshots.
- Determine surface type as product/data UI.
- Use product flow/design-system/data evidence more heavily than campaign galleries.
- Produce 3 directions with different density, component vocabulary, data grammar, and interaction model.
- Ask the user to choose before implementation.
- Use `data-ui-rubric.md`, typography review, responsive review, and artifact evidence before ready.

Failure signals:

- Justifies a dashboard with only Awwwards/Godly campaign references.
- Creates decorative charts with no axes, units, state, time window, action path, or exception priority.
- Uses `hero + cards`, shadcn-like card walls, or generic SaaS blue gradients.
