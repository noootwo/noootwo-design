# Direction Exploration

Direction work exists to explore meaningful possibility space, not to rename the same layout three times.

## When To Produce Directions

Produce 3 directions when:

- the user asks for a new design
- the request is ambiguous
- the work is a major redesign
- the brand or tone is known, but the execution path is still open
- the work is a redesign of an existing surface and the user did not explicitly ask for polish-only changes

You may skip directions only for:

- review-only requests
- extract-only requests
- narrow refinements inside an established screen or component family when the user explicitly asks to stay close to the current system

## Direction Format

Write results to `.noootwo/directions.md`.

Present the directions to the user before drafting. Do not silently choose one and continue.

Each direction must include:

- `Name`
- `Lineage`
- `Tone extreme`
- `One unforgettable thing`
- `Design philosophy`
- `Visual tone`
- `Typography`
- `Color discipline`
- `Grid and density`
- `Shape rules`
- `Component vocabulary`
- `Motion thesis`
- `Background/detail thesis`
- `Motion and interaction language`
- `Contrast strategy`
- `Image treatment`
- `Why it feels rare`
- `Why it fits`
- `Main risks`
- `What to borrow from the current system`
- `What must never happen`

Also include:

- `Recommended direction`
- `Why it is the default`
- `When to pivot away from it`

## Rules

- Directions must be materially different in type, structure, density, and mood
- Directions must also be materially different in motion language and component vocabulary
- Directions must also be materially different in background/detail treatment and first-screen structure
- Avoid 3 color swaps of the same layout
- Name directions like creative treatments or products, not generic labels like "Option A"
- If the current system is strong, keep at least one direction close to it
- Every direction must declare the type pairing and the density target explicitly
- Every direction must name the generic fallback patterns it is avoiding
- Every direction must be understandable and selectable by the user, not just by the model
- If 3 directions share the same opening structure or component skeleton, the exploration is invalid and must be redone
